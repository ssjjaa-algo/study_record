# 비트마스크의 정의

- **비트마스크(bitmask)** 는 여러 개의 상태를 **정수의 각 비트에 나누어 저장하는 방식**
- 예를 들어 `int`는 32bit이므로, 최대 32개의 원소에 대해 **포함 / 미포함**, **참 / 거짓**, **사용 / 미사용** 같은 상태를 한 번에 표현할 수 있다
- 모든 연산을 상수시간 내에 처리할 수 있음

## 장점

| 항목 | 설명                               |
| --- |----------------------------------|
| 표현 방식 | 여러 개의 boolean 상태를 정수 하나로 표현 가능   |
| 공간 효율 | boolean 배열보다 훨씬 적은 메모리로 저장 가능    |
| 연산 속도 | AND, OR, XOR, SHIFT 같은 비트 연산으로 매우 빠르게 처리 가능 |
| 부분집합 처리 | 부분집합 생성, 상태 압축 DP 등에 유리          |

## 비트마스크로 메모리를 효율적으로 쓰는 사례(하나씩 추가하며 정리)

### 1. boolean 배열 대신 정수 하나로 상태 저장

- 예를 들어 32개의 스위치가 켜져 있는지 꺼져 있는지를 저장한다고 하자

```java
boolean[] visited = new boolean[32]; // 32byte

int visited = 0; // 4byte
visited |= (1 << v); // 정점 방문
(visited & (1 << v)) != 0 // 정점 방문 여부 확인
visited &= ~(1 << v); // 정점 방문 해제
```

## 2. DP 테이블의 인덱스

- 이진수를 DP 테이블의 인덱스로 이용하는 방법
- DP 테이블의 크기는 2^n으로 나타내어 최적화를 취할 수 있음
- 단 n의 크기가 제곱으로 증가하기 때문에 n의 값이 크지 않아야 활용할 수 있음

# 테크닉

## 공집합과 꽉찬 집합

- 공집합은 0
- 꽉찬 집합 → (1 << n) - 1
    - (1 << 5)는 왼쪽 shift, 100000 (1 뒤에 0이 5개인 형태)
    - 100000의 -1 = **`011111`**

## 원소 추가

- num |= (1 << K);
    - K번째 bit를 키고 합집합
    - 10000 | 1000(1 << 3) = **`11000`**

## 원소의 포함 여부

- (num & (1 << K)) != 0
    - K번째 bit와의 교집합 여부 파악

## 원소의 삭제

- num &= ~(1 << K)
    - K번째 비트를 킨 것을 NOT 연산
        - 1000 → 0111
    - 그것과 AND 연산하면 K번째 bit는 0이 되고 나머지는 1과 연산하는 형태

## 원소의 토글

- num ^= (1 << p)

## 최소 원소 찾기 (최하위 비트 찾기)

- find = (num & -num)
    - num이 100이라 가정
    - -num은 2의 보수(NOT 연산한 결과 + 1)의 형태로 011 + 1 = 100이 된다
    - 100 & 100
    - 최하위 비트는 2^2 = 4
- 40 & -40
    - 101000 & (010111 + 1)
    - 101000 & 011000
    - 최하위 비트는 2^3 = 8

## 최소 원소 지우기

- num &= (num -1)
    - 1010 & 1001
    - num - 1은 최하위 비트를 끄고 그 밑의 비트를 전부 킨 형태
    - 따라서 &연산을 하면 위의 결과는 1000이 되고 결론적으로 최하위 비트인 2^1번째를 제거한 형태가 된다

## 모든 부분 집합 순회

- for (int subset = num; subset ≠ 0; subset = ((subset - 1) & num)
- 1씩 빼면 최하위 비트는 꺼지고 그 밑의 비트들이 전부 켜짐
- 이를 반복하여 모든 부분집합을 방문할 수 있다

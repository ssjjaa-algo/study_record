## Kubernetes에서의 Volume

- Pod에 Volume 연결해서 데이터 유지

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: random-number-generator
spec:
  containers:
    ...
    # host의 volume 마운트
      volumeMounts:
        - name: data-volume
          mountPath: /opt/out
  volumes:
    - name: data-volume
      hostPath:
        path: /data
        type: Directory
```

설명:

- `hostPath.path: /data` → 노드의 `/data` 디렉터리 사용
- `mountPath: /opt/out` → 컨테이너 내부 `/opt/out`에 마운트
- 컨테이너가 `/opt/out/number.txt`에 쓰는 데이터는 실제로 노드의 `/data`에 저장

---

## hostPath의 한계

- 멀티 노드 클러스터에서의 문제 → 다른 노드에 배치될 수 있다
    - 위의 예시로 Pod가 다른 노드에서 실행되면, 서로 다른 `/data` 디렉터리를 사용하게 된다.
    - 데이터 공유가 되지 않음
        - 이 이유로 멀티 노드 환경에서는 **`외부 공유 스토리지`**가 필요

---

## Kubernetes에서 지원하는 다양한 스토리지

- NFS
- Flocker
- Fibre Channel
- CephFS
- ScaleIO
- AWS EBS
- Azure Disk
- Azure File
- Google Persistent Disk

## AWS EBS 예시

```yaml
...
      volumeMounts:
        - mountPath: /data
          name: ebs-volume
  volumes:
    - name: ebs-volume
      awsElasticBlockStore:
        volumeID: vol-0abcd1234efgh5678
        fsType: ex4
```

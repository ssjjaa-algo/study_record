# 노드 네트워크 vs Pod 네트워크

- **`Node사이의 네트워크 통신이 원활하다 ≠ Pod 사이의 네트워킹이 원활하다`**
    - 따라서 Pod 또한 네트워킹 설정이 필요하다.
- 다음 질문에 답할 수 있어야 한다.
    - Pod는 어떤 방식으로 IP를 받는가?
    - Pod끼리는 어떻게 통신하는가?
    - 클러스터 내부에서 Pod의 서비스에 어떻게 접근하는가?
    - 외부에서는 어떻게 접근하는가?
- Kubernetes는 **내장된 Pod 네트워킹 솔루션을 제공하지 않음**
    - 대신 요구사항을 명확히 정의해두었음

---

# Kubernetes Pod 네트워킹 요구사항

1. 모든 Pod는 **고유한 IP 주소**를 가져야 한다.
2. 같은 노드에 있는 모든 Pod는 서로의 IP로 직접 통신 가능해야 한다.
3. 다른 노드에 있는 Pod와도 동일한 IP로 직접 통신 가능해야 한다.
4. NAT 설정 없이 통신 가능해야 한다.
- 어떤 IP 대역을 쓰는지는 중요하지 않다. 단, 자동 IP 할당과 연결성이 보장되면 된다.

---

# 예시

## 3개의 노드

- Node1 → 192.168.1.11
- Node2 → 192.168.1.12
- Node3 → 192.168.1.13
- 각 노드는 외부 네트워크에 연결되어 있음

### 각 노드에 Bridge 네트워크 생성

```bash
# 3개의 노드에서 각각 command 수행
ip link add v-net-0 type bridge
ip link set v-net-0 up
ip addr add 10.244.1.1/24 dev v-net-0 # 서브넷
```

- 각 노드에 서로 다른 서브넷을 할당
    - Node1 → 10.244.1.0/24
    - Node2 → 10.244.2.0/24
    - Node3 → 10.244.3.0/24
- 브리지 인터페이스에 IP 설정
    - Node1 → 10.244.1.1
    - Node2 → 10.244.2.1
    - Node3 → 10.244.3.1

# 컨테이너 생성 시 수행 작업

- 컨테이너 생성 시 다음을 수행하는 스크립트를 작성

```bash
ip link add ... # Create veth pair
ip link set ... # Attach veth pair

ip -n <namespace> addr add ... # Assign IP Address
ip -n <namespace> route add ...

ip -n <namespace> link set ... # Bring up interface

ip addr add 10.244.1.2/24 ...
ip route add default via 10.244.1.1
```

- 이제 같은 노드 내 Pod 간 통신 가능

---

# 다른 노드 Pod와 통신하기

- Pod 10.244.1.2 (Node1)
- Pod 10.244.2.2 (Node2)
- Node1은 10.244.2.2가 어디 있는지 모름
    - 따라서 Node1에 라우팅 추가

```bash
ip route add 10.244.2.0/24 via 192.168.1.12
```

- Node2에도 반대 방향을 설정해야 하고
- 모든 노드에 다른 노드의 Pod 네트워크에 대한 route 추가해야 함
    - 불편하므로 모든 노드가 라우터를 default gateway로 사용하게 한다면? → **`개선 지점`**
    - **모든 10.244.x.0 네트워크가 `하나의 큰 네트워크` (10.244.0.0/16)처럼 동작**

# CNI의 역할

- Kubernetes에게 "컨테이너 생성 시 이렇게 스크립트를 호출해라"라고 정의
    - 위에서 설정한 과정들을 실행하는 것
- 스크립트 작성자에게 "이런 형식으로 작성해야 한다"라고 정의
- 스크립트는 다음을 포함
    - ADD → 컨테이너 네트워크 추가
    - DEL → 컨테이너 네트워크 제거
    - IP 반환
    - 리소스 정리
- 컨테이너 런타임은 컨테이너를 생성할 책임을 가지고 있음
    - 컨테이너를 생성할 때 아래의 일을 수행함
        1. CNI 설정 확인

           —cni-conf-dir=/etc/cni/net.d

        2. CNI bin 디렉토리에서 플러그인 찾기

           —cni-bin-dir=/etc/cni/bin

        3. ADD 명령으로 호출
        4. 컨테이너 ID 및 네임스페이스 전달

           ./net-script.sh add <container> <namespace>

        5. 스크립트가 나머지를 처리

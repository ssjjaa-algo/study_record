## 1. 기본 개념

- **상황:** 사람이 곤충이 앉는 것을 막기 위해 몸에 퇴치제(**테인트**)를 뿌림
- **결과:** 냄새를 견디지 못하는 대부분의 곤충은 도망가지만, 그 냄새에 내성(**톨러레이션**)이 있는 특정 곤충은 사람에게 앉을 수 있음
    - **사람 = 노드(Node):** 테인트를 설정하여 포드의 접근을 밀어냄
    - **곤충 = 포드(Pod):** 톨러레이션을 설정하여 특정 테인트가 있는 노드에 배치될 수 있음
- **주의:** 보안 설정이 아니라, 스케줄링 시 특정 노드에 원치 않는 포드가 배치되지 않도록 **제한**하는 장치임

---

### 2. 작동 방식 및 시나리오

3개의 노드(1, 2, 3)와 4개의 포드(A, B, C, D)가 있는 클러스터 예시

1. **초기 상태:** 제약이 없으므로 스케줄러가 노드 전체에 포드를 골고루 분산 배치함
2. **테인트 설정:** 노드 1에 'blue'라는 테인트를 적용함
    - 톨러레이션이 없는 포드 A, B, C는 노드 1에 배치될 수 없어 노드 2나 3으로 밀려남
3. **톨러레이션 설정:** 포드 D에 'blue'에 대한 톨러레이션을 추가함
    - 이제 포드 D만 노드 1에 배치될 수 있게 됨

---

### 3. 테인트 설정 및 효과 (Taint Effects)

### **명령어 형식**

`kubectl taint nodes <노드이름> <키>=<값>:<효과>`

- 예: `kubectl taint nodes node1 app=blue:NoSchedule`

### **3가지 테인트 효과**

- 위 명령에서 app=blue:NoSchedule → NoSchedule이 아래의 3개의 값을 가질 수 있음
- **NoSchedule:** 톨러레이션이 없는 포드는 해당 노드에 **새로 스케줄링되지 않음**
- **PreferNoSchedule:** 시스템이 가급적 해당 노드에 포드를 배치하지 않으려 노력하지만, 자원이 부족하면 배치가 이루어질 수도 있음 (**보장되지 않음**)
- **NoExecute:** 새로운 포드 배치를 막을 뿐만 아니라, 이미 실행 중인 포드 중 톨러레이션이 없는 것들을 즉시 퇴출함

---

### 4. 포드에 톨러레이션 추가 (YAML 예시)

포드 정의서의 `spec` 섹션에 작성하며, **특이하게** **모든 값은 따옴표**(`""`)로 묶어야 함

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp-pod
spec:
  containers:
  - name: nginx-container
    image: nginx
  tolerations:
  - key: "app"
    operator: "Equal"
    value: "blue"
    effect: "NoSchedule" # NoSchedule / PreferNoSchedule / NoExecute
```

---

### 5. 한계점 및 주의사항

- **배치 보장의 부재:** 테인트와 톨러레이션은 노드가 포드를 **밀어내는 것**만 결정함. 포드 D가 'blue' **톨러레이션이 있다고 해서 반드시 노드 1에 가는 것은 아님 (제약이 없는 노드 2, 3으로 갈 수도 있음)**
- **해결책:** 포드를 특정 노드로 **끌어당기려면** `노드 어피니티(Node Affinity)`를 사용해야 함

---

### 6. Master Node

- 클러스터 구성 시 마스터 노드에는 일반적인 애플리케이션 포드가 스케줄링되지 않음
  - **이유:** 쿠버네티스 설치 시 마스터 노드에 자동으로 **NoSchedule 테인트**가 설정되기 때문
- **확인 방법:** `kubectl describe node <마스터노드이름>` 명령어를 실행하여 `Taints` 섹션을 확인하면 마스터 전용 테인트를 볼 수 있음

## 참고

- 기존에 있던 테인트를 지우는 것은 마지막에 -를 붙여야함

    ```bash
    kubectl taint nodes controlplane node-role.kubernetes.io/control-plane:NoSchedule-
    ```

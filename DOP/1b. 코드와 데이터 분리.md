- 코드와 데이터를 분리함으로써 시스템의 복잡도를 낮출 수 있다
    - 데이터가 코드에서 분리되면 시스템이 데이터 개체와 코드 모듈로 나뉘어 각각 독립적으로 생각할 수 있다
- 코드를 함수 안에 두는 방식으로 데이터에서 코드를 분리한다

## DOP 시스템의 두 부분

- 시스템의 복잡성과 경직성을 유발하는 주요 원인은 코드와 데이터가 객체에 함께 혼합되어 있다
    - DOP는 캡슐화에 반대한다
- DOP 원리는 언어에 구애받지 않는다
- **`데이터`  : 구성 요소만 가진 데이터 개체**
- **`코드`  : 무상태 함수**
    - 객체에서 상태와 캡슐화되는 대신 인자로 데이터 개체가 전달된다

## 데이터 개체

- 시스템에서 정보를 담고 있는 부분
- 도서관 관리 시스템을 예로
    - 책, 회원, 사용자, 사서, 제목, 저자, 도서 …
    - 시스템 요구 사항에서 명사와 명사구를 찾아보라
- 시스템의 데이터 개체를 알아내고 나서 다단 목록이나 마인드 맵을 사용해 상위 분류로 구분하자

## 코드 모듈

- 코드는 시스템의 기능을 식별하는 것
    - **시스템에 로그인**한다
    - 책을 **빌린다**
    - 도서 정보를 **검색**한다
    - 대출을 **금지**하거나 **해제**할 수 있다
    - 대출된 **책 현황을 조회**할 수 있다
- 시스템의 용도를 나열해본다
    - 도서관 시스템의 용도
        - 사용자 시스템 로그인
        - 도서 정보 검색
        - 책 추가
        - 회원 대출 금지
        - 회원 대출 금지 해제…
- 외부에 공개해야 할 api를 골라서 모듈 상자에 넣는다
    - Library가 가져야 할 함수의 모음
        - searchBook()
        - addBookItem()
        - blockMember()
        - unblockMember()…
- DOP 시스템의 코드 부분을 설계하는 첫 단계로 단일 모듈에 **`공개 기능`**을 모아놓는다

### OOP

```java
class Library {
	catalog
	userManagement

	getBookLendings(userId, memberId) {
		// 도서관의 상태는 this.catalog와 this.userManagement로	
	}
}
```

### DOP

```java
class Library {
	static getBookLendings(libraryData, userId, memberId)
	// 도서관의 상태는 libraryData에 보관되고 인자로 전달된다
	// DOP에서 모듈은 무상태이므로 처리할 도서관 데이터를 인자로 받아야한다
}
```

## 이해하기 쉬운 DOP 시스템

- 시스템이 코드 모듈과 데이터 개체로 나뉜다
- 관심사가 코드와 데이터로 분리된다
- DOP 모듈 간에는 사용 관계라는 유형의 관계만 있다
    - 한 모듈은 다른 모듈의 코드를 사용한다
    - 모듈 사이에 연관, 포함, 상속이란 개념은 없다

## 예시(맞는지 아직 모르겠음)

### 1. 기존 OOP 식으로 보면

- 도서관의 `Book` 객체가 자기 데이터와 동작을 같이 들고 있음.

```java
class Book {
    String title;
    String author;
    boolean borrowed;

    void borrow() {
        if (borrowed) throw new IllegalStateException();
        borrowed = true;
    }

    void returnBook() {
        borrowed = false;
    }
}
```

- 데이터: `title`, `author`, `borrowed`
- 코드: `borrow()`, `returnBook()`

가 **한 객체 안에 섞여 있다.**

### 2. DOP 식으로 보면

- 데이터는 그냥 데이터로 두고, 동작은 밖의 함수로 뺀다.

### 데이터 개체

```java
class BookData {
    String title;
    String author;
    boolean borrowed;
}
```

### 무상태 함수

```java
class BookService {
    static BookData borrow(BookData book) {
        if (book.borrowed) throw new IllegalStateException();

        BookData newBook = new BookData();
        newBook.title = book.title;
        newBook.author = book.author;
        newBook.borrowed = true;
        return newBook;
    }

    static BookData returnBook(BookData book) {
        BookData newBook = new BookData();
        newBook.title = book.title;
        newBook.author = book.author;
        newBook.borrowed = false;
        return newBook;
    }

```

- `BookData`는 **정보만 가진다**
- `BookService`는 **행동만 가진다**
- 함수는 자기 내부에 상태를 안 들고 있고, **입력값을 받아 결과를 돌려준다**

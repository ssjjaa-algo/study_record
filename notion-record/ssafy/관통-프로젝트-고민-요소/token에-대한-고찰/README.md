# token에 대한 고찰

## Session과 토큰의 보안 차이가 그렇게 큰가

- 차이는 정보가 드러나냐 아니냐
    - 키값이 드러나는지, 실제 내용물이 드러나는지의 차이다
- JWT token도 필요한 조금의 정보만 사용한다면 세션과 비슷하다고 본다
    - 우리 고찰 결과

## Refresh Token을 써야하는가

- Refresh token 쓰지 않고 Access token의 만료시간을 길게해서 쓰는건 어떨까?
- Refresh token이 가지는 장점은 브라우저를 다시 열었을 때 로그인이 되어있는 것
- 보안상 차이를 봐야 함
    - Access Token 시간을 길게하면,
        - 탈취 이전
            - 해커가 탈취를 시도할 시간이 많아짐.
        - 탈취당했을 때
            - 더 오래 쓸 수 있음.
    - 짧게 하면
        - 유저가 불편함
- 리프레시 토큰을 쓰면
    - httpOnly
    - 해커가 조작하기 더 힘들어짐.

## 관리 전략

- 리프레시토큰은 유저의 쿠키 httpOnly에 담아놓는다.
    - 서버에도 유저에게 하나 두고, 유저가 로그아웃하면 이 리프레시토큰을 서버에서 지운다.
- 로그아웃해도 엑세스토큰 삭제가 안됨.
    - 엑세스토큰을 블랙리스트 넣는다?
        - 토큰을 사용하는 이유가 없어짐.
    - 만료시간을 짧게 가져가는 것이 베스트로 보임.

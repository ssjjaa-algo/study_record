# 즉시 로딩 / 지연 로딩

# 지연 로딩(LAZY)

- 필요한 어떤 entity를 필요한 순간까지 사용하지 않다가 필요하면 호출
    - 프록시 특성으로 인해 처음에 target을 null로 해두고 필요할 때 세팅

# 즉시 로딩(EAGER)

- 바로 필요한거 다 가져오기.
- 여기에서 JPQL 이용 시 N + 1문제 발생 가능
    - **`JPQL은 SQL로 변환`**
    - SQL로 변환 → 만약 member 엔티티에 team 있다면 SQL을 어쩔 수 없이 2번 실행해야 하기 때문에.
- @XToOne 어노테이션은 기본이 즉시 로딩이어서 LAZY를 바꿔주자.

# 연관관계 매핑

# 연관관계의 주인과 mappedBy

## 객체의 양방향

- 단방향 연관관계 2개

## 테이블 양방향

- 외래키 하나로 두 테이블의 연관관계

### 둘 중 하나로 외래키 관리가 필요

## 연관관계의 주인

- 연관관계의 주인만이 외래 키를 관리(등록, 수정)
- 주인이 아닌 쪽은 읽기만 가능
- 주인은 mappedBy 속성 사용하지 않음
- 주인이 아니면 mappedBy 속성으로 주인 지정

### 주인을 선정하기

- 외래 키가 있는 곳을 주인으로 설정
- 다 쪽이 주인

💡 단방향 매핑으로 다 처리할 수 있다. 양방향은 그냥 객체 그래프 탐색용

</aside>

# 연관관계 매핑

## 다대일[N : 1]

- **`다 쪽에 외래키`**

## 일대다[1 : N]

- **`일 쪽에 외래키`**
- 엔티티가 관리하는 외래 키가 다른 테이블에 있어서 문제다.
- 연관관계 관리를 위한 추가 update sql
- 일대다 양방향은 공식적으로 없다.
    - JoinColumn(insertable=false, updatable=false)로 읽기 전용 필드를 양방향처럼 사용

## 일대일[1 : 1]

- 주 테이블에 외래키 단방향으로 사용하자.

## 다대다[N : M]

- 일대 다 다대 일 관계로 풀어서 사용

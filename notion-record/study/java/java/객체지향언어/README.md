# 객체지향언어

# 객체지향 이론의 기본 개념

- 실제 세계는 사물(객체)로 이루어져 있으며, 발생하는 모든 사건들은 사물 간의 상호작용

# 특징

- **코드의 재사용성**
- **코드의 관리 용이**
- 신뢰성 높은 프로그래밍 : **중복 제거**

# 클래스

- 객체를 정의해둔 것, 객체의 설계도, 틀

# 객체

- 실제로 존재하는 것, 사물 또는 개념

### 구성 요소

- 속성
    - 멤버변수, 특성, 필드, 상태
- 기능
    - 메서드, 함수, 행위

# 인스턴스

- **인스턴스화** : 클래스로부터 객체를 만드는 과정
- **인스턴스** : 클래스로부터 만들어진 객체

<aside>
💡 클래스 → (인스턴스화) → 인스턴스(객체)

</aside>

- 인스턴스는 참조변수를 통해서만 다룰 수 있고
- 참조변수의 타입은 인스턴스와 같아야 한다

### 인스턴스 변수

- 클래스 영역에 생성, 클래스의 인스턴스를 생성할 때마다 만들어진다.
- 독립된 저장 공간을 가짐

### 클래스 변수

- 인스턴스 변수 앞에 static
- 클래스 변수는 모든 인스턴스가 공통된 저장공간(변수)를 **공유**

### 지역 변수

- 메서드 내에 선언, 메서드 종료 시 소멸

# 메서드 사용 이유

- **높은 재사용성**
- **중복된 코드의 제거**
- **프로그램 구조화**

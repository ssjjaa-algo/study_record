# 클래스패스

- 클래스를 찾는 경로
    - OS에서 환경설정을 함으로서 지정 가능
        - 설정했을 때에는 해당 경로 아래에서만 한다.
    - 현재 디렉토리에 있는 모든 사용자 클래스 파일들을 지칭할때는 .(dot)을 사용
- Java Launcher의 클래스 찾기
    - **부트스트랩 클래스**에서 찾는다.
        - 자바 플랫폼을 구성하는 클래스 **`rt.jar`**(Object.class, String.class)에 포함되어 있는
    - 못찾으면 확장 클래스에서 찾는다.
        - jre안에 lib안에 ext
    - 여기서도 못찾으면 사용자가 정의한 클래스패스에서 찾아 메모리에 로딩
        - 없다면 ClassNotFound 오류 발생

<aside>
🧐 환경변수에서 클래스패스(classpath)를 별도로 지정하지 않았다면 현재 디렉토리에 있는 클래스를 찾는다. 하지만 c:\ classes등으로 지정했다면 기본적으로 현재 디렉토리에 참조할 클래스가 있어도 찾지 못한다. 별도로 클래스패스를 지정하는 경우에는 반드시 쩜(.)도 포함해야 한다.

</aside>

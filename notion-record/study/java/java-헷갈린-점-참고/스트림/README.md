# 스트림

# 데이터 소스의 추상화

- 코드의 재사용성 높아짐

# 특징

### 데이터 소스 변경하지 않는다

- 데이터를 읽기만 하고 변경은 하지 않음.
- 그러나 예시로 정렬된 결과를 컬렉션이나 배열에 담아서 반환할 수는 있다.

### 일회용

- Iterator와 동일
    - 한 번 사용하면 닫혀서 다시 사용할 수 없다.

### 내부 반복

## 연산

- 중간 연산과 최종 연산으로 분류
- 중간 연산
    - 연산 결과가 스트림인 연산, 스트림에 연속해서 중간 연산 가능
- 최종 연산
    - 연산 결과가 스트림이 아닌 연산. 스트림의 요소를 소모하므로 **`단 한 번`**만 가능

- 지연된 연산
    - 최종 연산이 수행되기 전까지 중간 연산이 수행되지 않는다.

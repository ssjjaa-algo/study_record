# 템플릿 콜백 패턴

전략 패턴에서 Context는 변하지 않는 템플릿 역할을 한다. 파라미터로 전략을 넘기는 경우, 변하는 부분이 파라미터(Strategy)가 되고 이 코드를 실행해서 처리한다. 이렇게 다른 코드의 인수로 넘겨주는 실행 가능한 코드를 콜백(Callback)이라 한다.

### 콜백 정의

다른 코드의 인수로서 넘겨주는 실행 가능한 코드, 콜백을 넘겨받는 코드는 콜백을 실행할 수도 있고 실행하지 않을 수도 있다.

코드가 호출(call)되는데 코드를 넘겨준 뒤(back)에서 실행되는 것.

클라이언트가 Context를 실행하는데, Strategy를 직접 실행하는 것이 아니라 Context를 실행할 때 Strategy를 넘겨주고 Context 뒤에서 Strategy를 실행하게 되는 그림

# AOP

# AOP 적용 방식

1. 컴파일 시점
2. 클래스 로딩 시점
3. 런타임 시점(프록시)

### 컴파일 시점

.java소스 컴파일 → .class를 만드는 시점에 부가 기능을 추가할 수 있다. AspectJ가 사용하는 특별한 컴파일러를 사용해야 하며, 이를 이용해 만들어진 .class파일을 디컴파일 해보면 로직이 들어가있음을 확인할 수 있다. 이렇게 원본 로직에 부가 기능 로직이 추가되는 것을 위빙(Weaving)이라 한다.

그러나 전용 컴파일러가 필요하다는 단점이 존재한다.

### 클래스 로딩 시점

.class파일은 JVM의 클래스 로더에 보관된다. 이 때 .class 파일의 bytecode를 조작해서 JVM에 올린다. JAVA는 .class를 JVM에 저장하기 전에 조작할 수 있는 기능을 제공하며, 이 시점에 Aspect를 적용하는 것을 로드 타임 위빙이라고 한다.

로드 타임 위빙은 자바를 실행할 때 클래스 로더 조작기를 지정해야 하는데, 이 과정이 번거롭고 어렵다고 한다.

### 런타임 시점

컴파일도, 클래스 로더에도 다 올라가서 이미 자바가 실행된 후를 말한다. 즉 자바의 main 메서드가 실행된 이후를 말한다. 자바 언어가 제공하는 범위 안에서 부가 기능을 적용해야 하며, 스프링 컨테이너에서는 프록시, DI, 빈 후처리기 등의 도움을 받아서 하는 것이다.

# AOP 적용 위치

적용 가능 지점을 join Point라 하며, 생성자 및 필드 지정자, static 메서드, 메서드에 적용이 다 가능하다.

AspectJ를 사용하는 경우 bytecode를 조작하기 때문에 모든 곳에 다 지정할 수 있다. 하지만 프록시 방식은 불가하다. 즉 **스프링 AOP는 오직 메서드에만 적용할 수 있다.** 프록시는 메서드 오버라이딩 개념으로 작동하기 때문이다. 또한 스프링 AOP는 스프링 빈에만 이용할 수 있다. 스프링은 AspectJ를 사용하는 것이 아니라 AspectJ의 문법만 차용하고 프록시 방식으로 작동한다.

# AOP 용어

1. join point

어드바이스가 적용될 수 있는 위치이며, AOP를 적용할 수 있는 모든 지점을 얘기한다. 스프링 AOP는 프록시 방식이기 때문에 항상 메서드 실행 지점이 적용 지점이 된다.

1. pointcut

join point 중에서 어드바이스가 적용될 위치를 선별한다. 스프링 AOP는 메서드 실행지점만 포인트컷이 가능하다.

1. target

어드바이스를 받는 객체를 말한다.

1. advice

부가 기능이다. Around, before, after같은 다양한 advice가 있다.

1. aspect

어드바이스 + 포인트 컷을 말한다. @Aspect

1. advisor

하나의 어드바이스와 하나의 포인트 컷으로 구성되는 것이다. 스프링 AOP에서만 사용되는 용어이다.

# 어드바이스 종류

1. Around

메서드 호출 전후에 수행, 가장 강력한 어드바이스, 조인 포인트 실행 여부, 반환값 변환, 예외 변환.. 다 가능

1. Before

조인 포인트 이전에 실행

1. After Returning

조인 포인트가 정상 완료 후 실행

1. After Throwing

메서드가 예외를 던지는 경우 실행

1. After

조인 포인트가 정상 또는 예외에 관계 없이 실행(finally)

구현의 순서를 천천히 따라가며, 실수하지 않게 짜기 위해 연습하기 좋은 문제라 생각.

# **`테스트 케이스 검증`**

## 구현 문제는 사소한 테케 하나하나가 중요하다

- 흰돌이나 검은돌이 출현한 위치에서 답이 가능한지 찾으러 간다. (1차 메서드 호출)
    - 방향은 왼쪽 → 오른쪽 순차 탐색을 하여서, 실제로 탐색은 동, 동남, 남, 남서만 한다.
    - 나머지 서, 북서, 북, 북동은 그 이전의 값이 자기 자신과 동일한지만 체크
        - 동일하면 탐색 중지
        - 안동일하면 탐색 가능성 확보
            - 탐색 가능성을 확보했다면
                - 선택한 방향으로 5개의 동일한 색깔의 돌이 있는지 확인한다
                - 5개의 돌이 동일하지 않다면 탐색 중지
                - 5개의 돌이 동일하다면 진짜 답이 되는지 확인하러 간다. (2차 메서드 호출)
                    - 6번째 돌이 나와 색깔이 같은 경우를 확인
                        - 같으면 답이 될 수 없음
                        - 다르면 답이므로 true 반환

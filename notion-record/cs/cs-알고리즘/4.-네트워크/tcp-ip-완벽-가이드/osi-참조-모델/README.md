# OSI 참조 모델

태그: 개념 정리

# 물리

- 데이터가 네트워크 인터페이스를 통해 물리적으로 전송되는 유일한 계층
- 0과 1 비트로 이루어지고
- 아무런 지식 없이 입력 비트를 받아 출력으로 보내기만 한다

## 오해하지 말아야 하는 것

- 물리계층이 NIC와 케이블에 불과하다고 주장하는 것은 틀린 말이다. 하드웨어 이외에도 여러 네트워크 기능을 정의하고 있다.
- 데이터 링크 계층과 연관이 깊어서 2계층에서 일어나는 에러 탐지, 정정 등이 1계층에서 일어날 수도 있다

## 기능

- 하드웨어 명세 정의
    - 케이블, 커넥터, NIC .. 하드웨어 장비 동작의 세부 사항
- 인코딩과 신호
    - 비트 데이터를 네트워크를 통해 전송하기 위한 신호로 인코딩하거나 변환
- 데이터 송수신
    - 데이터 인코딩 후 송수신
- 토폴로지와 물리 네트워크 설계
    - 여러 하드웨어 관련 네트워크 설계 문제를 다룬다

# 데이터 링크

- 유/무선 LAN 기술의 주요 기능이 위치하는 곳
    - 이더넷, 토큰 링, 802.11 등
    - 여기서 연결된 장비 모음들은 단순 네트워크로 간주
- 논리적 연결 제어 / 매체 접근 제어라는 하위 계층으로 분류
    - IEEE802 프로젝트에서 사용하는 구조에 근거
    - 분리했기 때문에 서로 다른 네트워크 기술 간의 상호 운용성 향상
- 브리지, 스위치, 브라우터 등

## 기능

- 논리적 연결 제어
    - 네트워크 계층에 서비스 제공, 데이터 링크 계층의 세부적인 사항을 감춰서 서로 다른 기술이 상위 계층과 쉽게 결합되도록 한다
- 매체 접근 제어
    - 장비가 네트워크 매체에 대한 접근을 통제하는데 사용하는 절차
    - 많은 네트워크가 공유 매체를 이용하기 때문에, 충돌을 피하기 위한 매체 관리 규칙 필요
        - CSMA/CD, 토큰링 등
- 데이터 프레이밍
    - Frame 캡슐화
- 주소지정
    - 정보에 특정 목적지 주소 붙임, MAC
- 에러 탐지와 처리
- 물리 계층 표준

# 네트워크

- 인터네트워크(서로 연결된 네트워크)가 어떻게 동작하는지 정의
- 원격 네트워크에 있는 컴퓨터에게 데이터를 전송할 수 있는 기능과 관련된 가장 낮은 계층
- 네트워크 계층에서부터 상위 계층의 추상적인 기능과, 데이터를 목적지로 보내는데 필요한 구체적인 작업이 구분되기 시작
- 라우팅 기능 수행, 최적의 경로를 찾기 위해 라우팅 프로토콜로 서로 통신

## 기능

- 논리적 주소 지정
    - 3계층 주소, 즉 IP
- 라우팅 네트워크
    - 최종 목적지를 파악하고, 그 목적지로 보내기 위해 패킷이 통과해야 할 다음 경로 결정
- 데이터그램 캡슐화
- 단편화 / 재조합
    - 패킷을 데이터링크 계층으로 보내야 한다.
    - 데이터링크 계층에서 송신할 수 있는 메시지의 길이를 제한하는 경우가 있다.
        - 이 때 단편화
        - 단편화된 데이터는 목적지 머신의 네트워크 계층에서 재조합
        - 이런 작업 수행하는 대표 프로토콜 : IP
- 에러 처리 / 진단
    - 네트워크 계층 프로토콜이 네트워크로 패킷을 보내기 위한 연결형 / 비연결형 서비스를 제공
        - 대부분 비연결형 서비스가 더 자주 일어남
        - TCP/IP에서 IP는 비연결형, TCP는 연결형
    

# 전송

- 세부적인 패킷 전송과 관련된 부분을 다루지 않음
- 장비 간 실제 데이터 이동을 직접 처리하지 않고 하위 계층 이용
- 서로 다른 컴퓨터에 있는 소프트웨어 애플리케이션 프로세스들이 통신하는데 필요한 기능 제공
    - 종단-종단 또는 호스트 간 전송

## 기능

- 프로세스 수준 주소 지정
    - 소프트웨어 프로그램을 구분하는데 쓰임
    - 이 기능으로 여러 소프트웨어 프로그램이 동시에 네트워크 계층 프로토콜 이용 가능
    - TCP / UDP 포트
        - TCP/IP 장비에서 동작하는 애플리케이션이 각각 별도로 처리되도록.
- 다중화 / 역다중화
    - 프로세스 수준 주소를 이용한다
    - 송신 전송 계층 → 데이터 다중화하여 데이터 스트림으로 결합
    - 수신 전송 계층 → 데이터그램 스트림 역다중화 → 적절한 수신 애플리케이션 프로세스로
- 단편화, 패키징,. 재조합
    - 데이터 단편화 후 목적지 머신에서 재조합 (네트워크 계층과 비슷)
    - 하위 네트워크 계층의 요구 사항에 맞게
- 연결 수립, 유지, 종료
- 승인과 재전송
    - 전송 계층은 안정적 데이터 전달을 보장하는 프로토콜을 구현하는 곳
        - 주로 승인과 재전송 타이머를 결합하여 사용
        - 타이머, 승인 정보에 따라 재전송 판단
- 흐름 제어
    - 통신에 참여하는 어떤 장비가 다른 장비에게 데이터 송신율을 낮추도록 요구
        - 데이터 수신자가 너무 많은 데이터를 처리하지 못하는 일을 예방

# 세션

- 장비가 세션을 수립하고 관리할 수 있도록
- 두 소프트웨어 애플리케이션 프로세스 간의 지속적인 논리적 연결
- 세션 생성, 유지, 종료하는데 필요한 수단 제공
- 세션 계층 도구들은 보통 상위 계층 프로토콜에 API 명령 집합으로 제공
    - 소켓 인터페이스 등

- 다른 계층보다 훨씬 적게 쓰이고, 많은 통신에서 필요로 하지 않음.

## 기능

- 번역
    - 서로 다른 데이터 표현 방법을 처리해준다
    - 차이가 있는 컴퓨터 간의 차이를 감춘다
- 압축
    - 데이터 압축
- 암호화
    - SSL 등

# 애플리케이션

- 네트워크 애플리케이션을 위한 계층
- 사용자가 네트워크를 통해 이루고자 하는 여러 작업을 실제로 수행하는 기능 구현
- 애플리케이션 자체가 아닌, OSI 모델 애플리케이션 계층은 사용자 애플리케이션이 이용할 수 있는 서비스를 제공한다.
- 일반적인 상황에서 컴퓨터에서 네트워크를 이용하는 프로그램을 사용한다는 것은
    - 애플리케이션 계층에 직접 접근하는 것
- HTTP, FTP, SMTP, Telnet 등

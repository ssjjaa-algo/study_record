# Transaction

## 정의

- 데이터베이스에서 수행되는 작업의 **`최소 단위`**
    - 입출금 → 하나의 묶음 형태로 작용해야함
- 무결성 유지, DB의 상태를 변화시키는 기능
- 하나 이상의 query를 포함하고, **`4가지 (ACID)`** 를 지켜야 한다.
    - **`원자성, 일관성, 고립성, 지속성`**

## 트랜잭션이 필요한 이유

- 데이터를 다룰 때 장애가 일어나는 경우 **`데이터를 복구하는 작업의 단위`**
- 여러 작업이 동시에 같은 데이터를 다룰 때 트랜잭션으로 분리, 오류가 발생하지 않게

### 4가지 규칙

- Atomicity(원자성)
    - transaction에 포함된 작업은 전부 수행되거나 전부 수행되지 않아야 한다
- Consistency(일관성)
    - transaction이 성공하면 언제나 일관성있는 데이터베이스 상태를 유지
- Isolation(고립성)
    - 각 transaction이 다른 transaction의 작업에 영향을 끼치지 않아야 한다.
    - 트랜잭션마다 **`독립적`**으로
        - 동시에 수행되는 트랜잭션이 동일한 데이터를 가지고 충돌하지 않도록 해야 함
            - 동시성 제어
- Durability(지속성)
    - 성공적으로 수행된 transaction은 데이터베이스에 영원히 반영한다

## 동시성 제어(concurrency control)

- 여러 개의 transaction이 한 데이터를 동시에 갱신할 때 어느 한 transaction이 무효화 → 갱신손실
- transaction이 동시에 수행될 때 일관성을 해치지 않도록 transaction의 데이터 접근을 제어하는 것

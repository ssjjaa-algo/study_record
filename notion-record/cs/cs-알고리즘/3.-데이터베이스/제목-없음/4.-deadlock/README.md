# 4. Deadlock

태그: Transaction

## 정의

- 여러 트랜잭션들이 각자 자신의 데이터에 대하여 lock을 획득한 상태에서 상대방의 데이터에 접근하고자 할 때 서로 무한정 대기하는 상태.

## 해결 방법

- 예방
    - 각 트랜잭션이 실행되기 전에 필요한 데이터를 모두 locking
    - transaction이 table a, table b에 대해 접근해야 한다면 둘을 모두 locking
        - locking을 해야되는 데이터가 많다면 트랜잭션의 **`병행성`** 보장이 어려움
- 회피
    - 자원을 할당할 때 timestamp 사용
- 탐지 / 회복
    - 데드락이 발생했을 때 이를 감지하고 회복시키는 방법

## 블로그 참고 (Lock의 해결 방안)

- 확실한 해결방법은 없다. 직관적인 나름의 차선책이 있는 느낌 정도.
- SQL 문장이 **`빠르게 실행`**되도록 설계 & 리팩토링
- 트랜잭션 시간을 **`가능한 짧게`** 정의
- 동일한 데이터를 동시에 update하는 작업을 피하도록 설계

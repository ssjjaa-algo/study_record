# 랜덤 액세스

태그 : http://www.gurubee.net/lecture/2230

# 랜덤 액세스가 발생한다는 것은 무엇인가

- 인덱스를 액세스하여 확인한 ROWID를 이용하여 테이블을 액세스하는 경우
- 조건을 만족하는 인덱스 값을 확인한 후 ROWID를 확인하여 ROWID 값으로 테이블을 액세스한다는 것
- I/O 한 번에 하나의 블록만 액세스

# 종류

- 확인 랜덤 액세스
    - WHERE 조건에 의해 발생하는 랜덤 액세스
    - 즉 **`WHERE 조건의 컬럼이 인덱스에 없어서`** 테이블을 액세스하는 랜덤 액세스
    - **추출되는 데이터의 건수가 감소할 수 있고, 버려지는 데이터가 존재할 수 있다.**

- 추출 랜덤 액세스
    - SELECT 절의 컬럼이 인덱스에 없는 경우
    - SELECT 절의 결과를 추출하기 위해 반드시 인덱스 액세스 후 테이블을 액세스해야함
    - **랜덤 액세스의 횟수와 추출되는 데이터의 양이 동일하다**

- 정렬 랜덤 액세스
    - ORDER BY 절 등에 사용된 컬럼이 인덱스에 존재하지 않아 테이블을 액세스하여 정렬을 수행하기 위해 데이터를 액세스하는 랜덤 액세스
    - **랜덤 액세스의 횟수와 추출되는 데이터의 양이 동일하다**

## 정렬 랜덤 액세스 감소

- 보통 ORDER BY 절의 컬럼을 추가하는 식으로 감소
- 즉 이거는 감소보다는 정렬 랜덤 액세스의 **`제거`**가 더 맞다.

## 추출 랜덤 액세스 감소

- 이는 SELECT 절의 모든 컬럼 인덱스를 추가해야 함
    - 매우 위험한 시도일 수 있어보임
- 어떠한 상황에 이를 고려해야 하는가
    - 매우 자주 사용하는 SQL
    - 하나의 인덱스로 많은 SQL의 추출 랜덤 액세스를 줄일 수 있는 경우
    - 인라인 뷰를 통해 데이터가 감소하는 경우 **ROWID를 이용하여 추출 랜덤 액세스 감소를 고려**

### 3번째 경우 (ROWID 이용 추출 랜덤 액세스 감소)

- index : (카드번호 + 거래일자)

```sql
SELECT 카드번호, 거래일자, 거래구분, 사용액
  FROM
     ( SELECT 카드번호, 거래일자, 거래구분, 사용액
         FROM 거래내역
        WHERE 카드번호 = '111'
        ORDER BY 거래일자
     )
 WHERE ROWNUM <= 5;
```

- 위의 쿼리는 최종 결과 집합은 5개이지만 SELECT의 다른 컬럼을 가져오기 위해 추출 랜덤 액세스만 1000번 발생

```sql
SELECT 카드번호, 거래일자, 거래구분, 사용액
  FROM
     ( SELECT ROWID RID
         FROM 거래내역
        WHERE 카드번호 = '111'
        ORDER BY 거래일자
     ) A,
       거래내역 B
 WHERE A.RID = B.ROWID
   AND ROWNUM <= 5;
```

- A 인라인 뷰는 ROWID 값이 A 인덱스에 존재하기에 커버링 인덱스 가능
    - 따라서 모든 랜덤 액세스는 발생하지 않고
    - 거래내역 테이블에 5번의 랜덤 액세스만 발생

💡

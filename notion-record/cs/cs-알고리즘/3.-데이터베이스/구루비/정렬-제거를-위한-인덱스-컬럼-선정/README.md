# 정렬 제거를 위한 인덱스 컬럼 선정

태그 : http://www.gurubee.net/lecture/2236

```sql
SELECT 카드번호, 사용액
  FROM 거래내역
 WHERE 카드번호 = '111'
 ORDER BY 거래일자;
```

- 위의 SQL은 (카드번호 + 거래일자)로 가능

```sql
SELECT 카드번호, 사용액
  FROM 거래내역
 WHERE 카드번호 = '111'
   AND 거래일자 BETWEEN '20080801' AND '20080830'
 ORDER BY 승인번호;
```

- (카드번호 + 거래일자 + 승인번호)
    - 정렬 생략 불가
        
        ```sql
        카드번호  거래일자    승인번호
        111     20080801   1
        111     20080801   2
        111     20080802   1
        111     20080802   3
        ```
        
    - 인덱스를 이용하면 위와 같이 되므로 승인번호 순의 정렬이 아님
- (카드번호 + 승인번호 + [거래일자])
    - 이 경우 승인번호가 정렬되있는 상태에서 거래일자를 스캔하게 된다.
    - 카드번호+승인번호+거래일자 인덱스의 거래일자 컬럼은 **처리 범위를 감소시키지는 못하며 확인 랜덤 엑세스만을 제거하게 된다**
- 처리 범위를 "줄인다"는 것은 **`쿼리에서 조건이 인덱스의 탐색 단계에서 바로 적용되는지 여부`**에 따라 결정
- **카드번호 조건**은 탐색 단계에서 인덱스 범위를 직접 줄이므로, 처리 범위를 줄이는 데 기여
- **거래일자 조건**은 탐색 단계에서 적용되지 않고 **후처리로 필터링,** **처리 범위를 줄인다고 보지 못함**

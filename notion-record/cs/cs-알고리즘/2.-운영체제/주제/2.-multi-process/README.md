# 2. Multi process

면접 대답

시분할 시스템 → context → PCB → context switching, process의 메모리 영역 설명

## 개념

- 2개 이상의 프로세스가 동시에 실행되는 것, 이 때 process들은 CPU와 메모리를 공유한다.
- memory의 경우 여러 process들이 각자의 memory 영역을 차지하여 동시에 적재
- 하나의 CPU는 매 순간 하나의 process만 연산할 수 있는데, CPU의 처리 속도가 너무 빨라서 아주 짧은 시간동안 여러 process들이 CPU에서 번갈아 실행되기 때문에 여러 프로그램이 동시에 실행되는 것처럼 보임
    - CPU의 작업시간을 여러 process들이 조금씩 나누어 쓴다 → **`시분할 시스템(time sharing system)`**
    - **`동시성`**(concurrency)와 **`병렬성`**(parallelism)을 의미함
        - 동시성
            - CPU core가 1개일 때, 여러 process를 짧은 시간동안 번갈아가면서 시분할 시스템으로 실행
            - 얘는 너무 빨라서 **`동시처럼 보일 수 있음`**
        - 병렬성
            - CPU core가 여러개일 때, 각각의 core가 각각의 process를 연산함으로써 process가 동시에 실행되는 것
            - 얘는 **`실제로 동시에 실행`**
            

## 특징

- 메모리 관리
    - 메모리에 올라와 있는 프로세스 각각에 대해 침범하지 못하도록 운영체제가 관리
- CPU의 연산과  PC register
    - PC register가 가리키고 있는 명령어를 CPU가 읽어서 진행
    - process1을 가리키다가, process2를 가리키다가. 왔다갔다 하며 명령어를 읽어들이고 연산을 한다
- Context
    - 시분할 시스템에서 한 process가 짧은 시간동안 CPU를 점유하여 일정 부분의 명령을 수행 후, 다른 process에게 넘기고, 다시 차례가 되면 CPU를 점유하여 명령을 수행한다
    - 그래서 이전에 어디까지 명령을 수행했고, register에는 어떤 값이 저장되어 있었는지에 대한 정보가 필요하다 → 이러한 것에 대한 총체적인 정보가 **`context`**
    - context는 **`PCB(Process Control Block)`**에 저장
- PCB
    - 운영체제가 프로세스를 표현한 자료구조
    - 프로세스의 중요한 정보가 포함되어 있기 때문에 보호된 메모리 영역 안에 저장
    - 프로세스 스케줄링을 위해 프로세스에 관한 정보를 저장하는 **`임시 저장소`**
    
    ![Untitled](../../../../../assets/untitled-373.png)
    
    - 포인터 : 프로세스의 현재 위치 저장
    - 프로세스 상태(state) : 생성, 준비, 실행, 대기, 종료 ..
    - 프로세스 아이디 : 프로세스 식별자
    - 프로그램 카운터 : 프로세스를 위해 실행될 다음 명령어의 주소 저장
    - 레지스터 : 누산기, 베이스, 레지스터 및 범용 레지스터를 포함하는 CPU 레지스터에 있는 정보
    - 메모리 제한 : 운영체제에서 사용하는 메모리 관리 시스템에 대한 정보
    - 열린 파일 목록 : 프로세스를 위해 열린 파일 목록

- 프로세스의 상태(state)
    - 실행(running), 준비(ready), 봉쇄(wait, sleep, blocked) 세 가지 상태로 구분
        - 실행
            - 프로세스가 CPU를 점유하고 명령을 수행중인 상태
        - 준비
            - CPU만 할당받으면 즉시 명령을 수행할 수 있도록 준비된 상태
        - 봉쇄
            - CPU를 할당받아도 명령을 실행할 수 없는 상태

- Context switch
    - 한 프로세스에서 다른 프로세스로 CPU 제어권을 넘겨주는 것
    - 이 때 이전의 프로세스의 상태를 PCB에 저장하여 보관하고
    - 새로운 프로세스의 PCB를 읽어서 보관된 상태를 복구하는 작업이 이루어진다

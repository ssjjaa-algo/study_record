# 8. Deadlock

## 개념

- 둘 이상의 thread가 각기 다른 thread에 대한 자원을 대기할 때 **`무한 대기`**에 빠지는 현상

## 데드락이 일어나는 조건 4가지

- 상호 배제
    - 동시에 한 쓰레드만 자원을 점유할 수 있는 상황
    - 다른 쓰레드가 자원을 사용하려면 자원이 방출될 때까지 대기
- 점유 대기
    - 쓰레드가 자원을 보유한 상태에서 다른 쓰레드가 보유한 자원을 추가로 대기하는 상태
- 순환 대기
    - 대기중인 쓰레드들이 순환 형태로 자원을 대기하고 있는 상태
- 비선점
    - 다른 쓰레드가 선점한 자원을 강제로 선점할 수 없는 상황
    - 자원을 점유하고 있는 쓰레드에 의해서만 자원이 방출

## 데드락 문제를 해결하는 방법

- 무시
    - 데드락 발생 확률이 낮은 시스템에서 데드락을 **`무시해버리기`**
        - 시스템 성능 저하가 없다는 장점.
        - 현대에서는 잘 발생하지 않고 해결 비용이 많이 들어 주로 사용
- 예방
    - 교착 상태의 4가지 발생 조건 중 하나가 성립되지 않게 하는 것
        - 현실적으로 순환 대기를 예방하는 것이 가능
            - 자원 사용의 효율성이 떨어지고 비용이 크다
- 회피
    - 쓰레드가 앞으로 자원을 어떻게 요청할지에 대한 정보를 통해 순환대기 상태가 일어나지 않도록 하는 것 (미리 예측)
        - 자원 할당 그래프 알고리즘, 은행원 알고리즘 등
- 탐지 회복
    - 시스템 검사를 통해 데드락 발생을 탐지하고, 데드락 발생이 탐지되었으면 회복 시키는 것
        - 자원 사용의 효율성이 떨어지고 비용이 크다

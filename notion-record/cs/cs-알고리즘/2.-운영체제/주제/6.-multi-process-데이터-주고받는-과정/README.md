# 6.  multi process 데이터 주고받는 과정

태그: IPC, kernel, message, shared

## multi process 환경에서 process 간에 데이터를 어떻게 주고 받음?

- 원칙
    - 프로세스는 **`독립적인 주소 공간`**을 가져서 다른 프로세스의 주소 공간 참조 불가
- 경우에 따라 프로세스 간의 자원 접근을 위한 매커니즘 제공
    - **`IPC(Inter Process Comunication)`**
        - 파이프, 파일, 소켓, 공유메모리 등

## IPC

- **`다른 프로세스와의 통신`**을 가능하게 해주기 위해.
- 공유메모리(shared memory)
    - 과정
        - 프로세스들이 **`주소 공간의 일부`**를 공유
        - 공유한 메모리 영역에 읽기/쓰기를 해서 통신을 수행
        - 프로세스가 **`공유 메모리 할당을 kernel에 요청`**
        - kernel은 **`해당 process에 메모리 공간을 할당`**
        - 공유 메모리 영역이 구축된 이후는 모든 접근이 일반적인 메모리 접근으로 취급
        - 이제부터는 커널 도움 없이도 메모리 영역에 접근 가능
            - **`속도가 빠르다`**
    - 단점
        - 일관성 문제
            - **`동시에`** 같은 메모리 위치에 접근
            - 커널이 관여하지 않기 때문에 **`프로세스들끼리 동기화 문제를 책임져야함`**
    
- 메시지 전달(message)
    - 시스템 콜을 사용하여 구현
        - **`커널을 통해`**
            - send message
            - receive message
        - process1이 커널에 message를 보냄
        - 커널이 process2에 메시지를 보냄
        - **`충돌을 회피할 필요가 없다`**
        - 적은 양의 데이터를 교환하는데 유용
    
    - 단점
        - 메모리 공유보다 속도가 느림

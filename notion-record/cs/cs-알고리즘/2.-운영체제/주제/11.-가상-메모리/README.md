# 11. 가상 메모리

## 개념

- 프로세스 전체가 메모리에 올라와있지 않더라도 실행할 수 있게 하는 방법
    - 사용자 프로그램이 물리적 메모리보다 커져도 실행이 가능하다는 장점

## 가상 메모리

- 실제 물리 메모리 개념과 개발자 입장의 논리 메모리 개념을 분리한 것
- 개발자가 메모리 크기에 염려할 필요없이 쉽게 프로그램을 작성할 수 있게 해줌
- 운영체제는 가상 메모리 기법을 통해 프로그램의 논리적주소 영역에서 **`필요한 부분만 물리적 메모리에 적재`**, 직접적으로 **`필요하지 않은 메모리 공간은 디스크(Swap) 영역`**에 저장
    - 사용하는 것만 물리적 메모리에 올린다

## 요구 페이징

- **`당장 사용될 주소 공간`**을 page 단위로 적재한다
    - 메모리 효율적 사용 가능
    - 프로세스 전체를 메모리에 적재할 필요 없다
        - 입출력 오버헤드 감소
- 특정 page에 대해 **`CPU 요청이 들어와야`** 메모리에 적재한다.
- **`유효 / 무효 bit`**
    - 각 페이지가 메모리에 존재하는 지를 check

## 페이지 부재

- **`CPU가 무효 bit로 표시된 곳에 액세스`** 하는 경우
- 무효 페이지에 접근을 하면 주소 변환을 담당하는 하드웨어인 MMU가 페이지 부재 트랩을 발생시킨다.
    - 순서
        - CPU가 페이지 a를 참조한다
        - 페이지 테이블에서 a가 무효 상태임을 확인한다
        - 커널에 page fault trap이 발생함을 알림
        - 디스크 영역에서 페이지 a를 물리 메모리에 적재

## 페이지 교체 알고리즘

- page falut 발생 시 요청한 page를 디스크에서 가져오는데, 이 때 물리적 메모리 공간 부족할 수 있음.
- 이 경우에는 메모리에 올라와 있는 페이지 중 일부를 디스크로 옮겨서 메모리를 확보 —> 페이지 교체
- 어떤 페이지를 교체할 것이냐를 결정하는 것이 페이지 교체 알고리즘
- 교체 알고리즘은 page fault가 최대한 적게 나도록 해줘야함
    - FIFO
        - 메모리에 가장 오랫동안 올라와 있던 페이지 교체
    - 최적 페이지 교체
        - **`앞으로`** 오랫동안 사용되지 않을 페이지 교체 —> 어렵다.
    - LRU(Least Recently Used)
        - 가장 오랫동안 사용되지 않은 페이지 교체
    - LFU(Least Frequently User)
        - 참조횟수가 가장 적은 페이지를 교체

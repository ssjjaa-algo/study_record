# ***OSI 7계층***

- Why?
    - 컴퓨터마다 시스템이 달라 유지보수가 어려우니 표준 모델을 만들자

- PDU(Process Data Unit)
    - 각 계층에서 전송되는 단위
        - 2계층 Frame
        - 3계층 Packet
        - 4계층 Segment

- 물리 계층 (1계층, Physical Layer)
    - 최하위 계층
    - 상위 계층에서 전송된 데이터 → 물리 매체 (허브, 케이블 등)을 통해 전기전 신호 전송
    - 아날로그 신호를 전송하는 것만 신경쓴다
    - 그니까 에러나 오류는 안잡고 전송에만 신경쓰는 놈
    - 장비 : 허브, 리피터 등

- 링크 계층 (2계층, Link Layer)
    - 네트워크 기기들 사이의 데이터 전송 역할
    - 오류없는 데이터 전송을 위한 ‘Frame’ 구성 → 물리계층 전송
    - 프레임에는 시작과 끝단을 알리는 신호가 붙어서 간다
        - 여러 개의 데이터를 동시에 보냈다고 가정했을 때, 이들을 구분지어 줘야함
        - 프레임의 시작(0000), 끝(1111)라고 가정하면 이 신호를 보고 파악한다.
    - 네트워크계층에서 주소와 제어정보를 헤더와 테일에 추가
    - 장비 : 브리지, 스위치 등

- 네트워크 계층(3계층, Network Layer)
    - 라우팅 → 경로 설정,
    - 라우터 : **데이터 패킷을 의도한 IP 주소로 전달하여 이러한 네트워크 간의 트래픽을 관리하고, 여러 장치가 동일한 인터넷 연결을 사용할 수 있도록 하는 등**, 두 가지 주요 기능을 제공합니다.
    - 내가 가고 싶은 곳으로 안내해주는 놈.
    - 목적지 주소를 알고있어야 가능 (DNS를 알고 접근하기 때문에 이미 알고있음)
    - 목적지까지 성공적이고 효과적으로 전달하자
    - 장비 : 라우터, L3 스위치(스위치 + 라우팅)

- 전송 계층(4계층, Transport Layer)
    - End-to-End 간 제어와 에러 관리
    - 패킷의 전송이 유효한지 확인
    - 패킷의 재전송을 보장
    - Segment 사용, 세그먼트는 포트(port)를 가지고 있다.
        - 포트 이용하여 응용 프로그램이나 서비스 식별 역할
    - 장비 : 게이트웨이, L4 스위치

- 세션 계층(5계층, Session Layer)
    - 포트 번호 기반 연결
    - 장치 간의 상호작용을 설정, 유지, 동기화
    - 너랑 나랑 어떤 식으로 통신할거니? 전화, 카톡 등.. ***연결***.
    - 근데 여기서 반이중 방식으로 통신할거냐, 무선 통신 할거냐 등을 정할 수 있다.

- 표현 계층(6계층, Presentation Layer)
    - 데이터를 응용 프로그램이나 네트워크와 호환될 수 있는 형식으로 변환하는 역할
    - 데이터의 표현 방식과 구조를 조정하여 다른 시스템과의 호환성을 보장하고 데이터 전송의 안정성과 효율성을 유지하는 역할
    - 전송의 안정성과 효율성을 유지한다? : 암호화 등을 해주겠다.
    - 암호화, 표현, 처리.. 다 표현계층에서.

- 응용 계층(7계층, Application Layer)
    - 사용자와 바로 연결
    - 최종 사용자에게 “가장 가까운” 계층
    - 사용자에게 정보를 입력받아 하위로 전달
    - 하위 계층에서 전송한 데이터를 사용자에게 전달
    - 나는 너와 통신을 할 건데 뭐 이용해서 통신? (카카오톡, Twitter, 버디버디… 등등 … )

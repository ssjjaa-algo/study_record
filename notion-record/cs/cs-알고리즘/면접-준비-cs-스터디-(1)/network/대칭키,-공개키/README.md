# 대칭키, 공개키

날짜: 2023년 3월 31일
사람: 유영

## [대칭키](http://wiki.hash.kr/index.php/%EB%8C%80%EC%B9%AD%ED%82%A4)

> 암호화와 복호화에 사용하는 키 값이 **동일**한 암호화 방식
> 
- 송 · 수신자 모두 같은 키를 가지고 있어야 한다.

### 종류

1. 스트림 암호
    
    **연속**적인 **비트** 혹은 **바이트**를 계속 입력 받아 그에 대응하는 암호화 비트 혹은 바이트를 생성하는 방식
    
2. 블록 암호 (ECB 모드)
    
    정해진 한 **단위**(블록)를 입력 받아 그에 대응하는 암호화 블록을 생성하는 방식
    

### 대표적인 예시

1. [AES](http://wiki.hash.kr/index.php/AES) (Advanced Encryption Standard)
    1. 고급 암호화 표준의 블록 암호 알고리즘 (안정성 문제 제기)
    2. 현재까지 알려진 대칭키 암호화 알고리즘 중 가장 안전한 것 중 하나
    3. 블록의 크기는 128 bits, 키의 크기는 128 bits, 192 bits, 256 bits를 지원
2. [SEED](http://wiki.hash.kr/index.php/SEED)
    1. 개인 정보 보호를 위해 한국 정보보호센터(KISA)에 의해 개발된  블록 암호 알고리즘 (안정성 문제 제기)
    2. ActiveX로 배포된다. (호환성 문제 제기)
    3. 블록의 크기는 128 bits, 키의 크기는 128 bits, 192 bits, 256 bits를 지원

### 장 · 단점

- 내부 구조가 간단한 **치환**(대치)과 **전치**(뒤섞기)의 조합으로 되어있기 때문에 공개키 암호화 방식에 비해 **계산 속도**가 빠르다.
- 키를 교환하는 과정이 어렵고 위험하다.
    - 해커가 키를 **전달**하는 과정에서 데이터를 가로챌 경우 가로챈 **키 하나**만으로 암 · 복호화가 가능해진다.
- 사용자 당 **하나**의 키를 가지고 있기 때문에 많은 사람과 data 교환 시 **키 관리**가 복잡하다.

## [공개키](http://wiki.hash.kr/index.php/%EA%B3%B5%EA%B0%9C%ED%82%A4)

> 암호화와 복호화에 사용하는 키가 **서로 다른** 암호화 방식
> 
- 암호화엔 모두가 볼 수 있는 **공개키**, 복호화엔 개인만 알고 있는 **비밀키**를 사용한다.
- 공개키를 알아도 복호화가 어려운 이유 : **Trap door one way function**에 기반을 두고 있기 때문

<aside>
👉 **Trap door one way function**이란?

한 방향으로는 계산이 쉽지만 다른 방향으로는 계산이 어렵게 만든 수식 (소인수분해, 이산대수 등)

</aside>

### 종류

1. [RSA](http://wiki.hash.kr/index.php/RSA)
    1. R 씨, S 씨, A 씨가 개발한 알고리즘
    2. 수 백 개의 소프트웨어와 연동이 가능하며, 국제표준화기구(ISO)를 비롯한 여러 국제기구에 암호표준으로 제안되어 있다.
    3. 두 개의 140자리 이상인 엄청나게 큰 소수의 곱과 추가연산을 통해 개인키와 공개키를 구성
    4. 동작 원리는 매우 복잡하다. 우리가 알면 세계 제일 해커겠지요. 
    5. 보안은 개인키의 암호를 해독하려면 슈퍼컴퓨터로도 1만 년 이상이 소요. 하지만 많은 계산량으로 인해 휴대용 단말기에선 사용하기 어렵다.

### 장 · 단점

- 복호화에 필요한 키를 전달하는 작업을 하지 않으므로 대칭키 방식에 비해 **안전**하다.
- 많은 사용자와 data를 교환하더라도 **같은** 공개키를 공유, **같은** 비밀키로 복호화하면 되므로 **키 관리**가 비교적 쉽다.
- 키 길이가 길고 , 공개키를 이용한 복호화를 방지하기 위한 복잡한 **수학적 연산**을 이용하기 때문에 **계산 속도**가 느리다.

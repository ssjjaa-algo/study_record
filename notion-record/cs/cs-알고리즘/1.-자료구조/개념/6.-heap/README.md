# 6. Heap

- 우선순위 큐와 일치
- Heap은 **`완전이진트리의 특징`**을 가지는 자료구조.

- Heap은 2가지가 존재
    - **`MaxHeap`**
        - 각 노드에 저장된 값은 자식 노드에 저장된 값보다 크거나 같다
        - root 노드에 있는 값은 가장 큰 값
    - **`MinHeap`**
        - 각 노드에 저장된 값은 자식 노드에 저장된 값보다 작거나 같다
        - root 노드에 있는 값은 가장 작은 값

- Heap은 트리이지만 Array로 구현한다
    - 새로운 노드 추가는 힙의 **`마지막 위치`**에 추가
    - Array로 구현해야 이 과정이 편하다
    - array의 index로 부모/자식 간의 관계를 확인한다 ( **`완전 이진 트리이기 때문에`** 가능)
        - 왼쪽 자식은 (idx * 2)
        - 오른쪽 자식은  (idx * 2) + 1
        - 부모 노드는 (idx / 2)
    
- Heap의 push, pop에 걸리는 시간복잡도는  **`O(logN)`**
- MaxHeap으로 예시
    - push
        - 힙의 맨 마지막 노드에 추가
            - 부모 노드의 값이 더 작다면 swap
                - 계속 반복
                - 최대 트리의 높이만큼 반복.
    
    - pop
        - 힙의 최댓값을 뽑는다. (root 노드)
        - 맨 마지막에 있는 노드를 루트로 옮긴다
        - MaxHeap의 조건이 부합하도록 자식들과 비교한다
            - 자식들 중에 큰 값과 Swap
            - child node가 더 크지 않을 떄까지 계속 반복
                - 최대 트리의 높이만큼 반복

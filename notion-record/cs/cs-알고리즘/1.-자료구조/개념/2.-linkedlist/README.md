# 2. LinkedList

- Node라는 구조체로 이루어져있는 자료구조
    - Node에는 데이터 값과, **`다음 Node의 주소 값`**을 저장
    - **`메모리상에서는 비연속적`**으로 저장
    - 하지만 각각의 Node가 저장한 주소를 기반으로 접근하기 때문에 **`논리적인 연속성`**

- tree, graph 등의 다른 자료구조를 구현할 때 자주 쓰이는 기본 자료구조

- 데이터가 추가되는 시점에 메모리를 할당한다
    - **`메모리의 효율적인 사용`**이 가능하다.
    - 다음 Node의 주소값을 저장하기 때문에 필드를 1개 더 쓴다는 의미로 데이터 하나당 차지하는 부피는 더 크다

|  | Linked list |
| --- | --- |
| access | $O(n)$ |
| search | $O(n)$ |
| insertion | $O(1)$ |
| deletion | $O(1)$ |
- 삽입 삭제는 단순히 **`가리키는 노드의 주소를 변경하`**기만 하면 되어 O(1)
    - 그러나 실제로는 O(n)
        - **`삽입하거나 삭제하는 지점`**까지 찾아가야 하기 때문에 O(n)을 소요해야 한다

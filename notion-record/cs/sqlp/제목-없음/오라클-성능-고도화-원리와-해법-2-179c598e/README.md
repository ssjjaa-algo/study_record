# 오라클 성능 고도화 원리와 해법 2

텍스트: 병렬 처리

### Query Coordinator와 병렬 서버 프로세스

- QC는 작업 반장의 느낌
    - 일단 사용자가 지정한 병렬도와 오퍼레이션 종류에 따라 하나 또는 두 개의 병렬 서버 집합 생성
        - 서버 풀로부터 필요한 만큼 서버 프로세스 확보하고 부족하면 새로 생성
    - 각 병렬 서버에게 작업을 할당하고 쳐다보고 있음
    - 병렬로 처리하도록 사용자가 지시하지 않은 테이블은 QC가 직접 처리
    - 병렬 서버로부터의 산출물 통합

### Intra-Operation Parallelism과 Inter-Operation Parallelism

- 명함을 분배하는 것
    - 분배팀과 정렬팀으로 나눈다.
    - 분배팀이 QC로부터 할당받은 작업을 고객명에 따라 정해진 담당자에게 건넨다.
- 서로 배타적인 범위를 독립적으로 동시에 처리(Intra)
    - 분배 팀이 자신에게 할당받은 것을 상대편에게 분배한다.
    - 정렬 팀이 1~3, 4~6, 7~9.. 처럼 맡은 범위에 대한 것만 정렬한다.
- 정렬된 결과를 QC에게 전송하는 작업을 병렬로 동시에 진행하는 것(Inter)
    - 정렬 팀이 결과를 QC에게 준다.

### 테이블 큐

- Intra 방식은 한 병렬 서버 집합(Server Set)에 속한 여러 프로세스가 처리 범위를 달리하면서 병렬로 작업을 진행하는 것 → 프로세스간 통신이 발생하지 않음
- Inter는 프로세스 간 통신이 발생하고, 이를 위한 통신 채널이 필요함 → 이 채널이 테이블 큐

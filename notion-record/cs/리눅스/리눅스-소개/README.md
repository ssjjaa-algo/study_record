### **1. 병렬 실행의 "환상"**

- **"여러 사용자나 프로세스가 병렬로 실행된다는 환상"**:
    - 실제로 컴퓨터 시스템에서는 CPU, 메모리, 네트워크 등 **물리적 리소스가 제한적**입니다.
    - 그러나 운영체제는 사용자가 마치 **독립된 환경에서 자신의 프로세스만 실행되고 있는 것처럼 보이게** 만드는 기능을 제공합니다.
    - 이는 운영체제의 **가상화 기술**과 **리소스 격리 기술** 덕분에 가능하며, 이를 통해 리소스를 공유하는 여러 사용자나 프로세스가 서로 간섭하지 않는 것처럼 보이게 됩니다.

---

### **2. 리소스에 대한 (제한된) 가시성**

- **"리소스에 대한 제한된 가시성"**:
    - 리눅스는 사용자나 프로세스에게 리소스(파일 시스템, 네트워크 인터페이스, 프로세스 목록 등)에 대해 **전체적인 뷰가 아닌 제한된 뷰**를 제공합니다.
    - 예를 들어, 프로세스가 실행 중인 컨테이너에서는 **자신만의 독립된 파일 시스템과 네트워크 인터페이스**만 볼 수 있고, 시스템의 다른 부분은 보이지 않습니다.
    - 이를 통해 사용자는 자신만의 환경에서 작업을 하고 있다고 생각할 수 있습니다.

### **3. 네임스페이스를 통한 로컬 보기 제공**

- **"리눅스에서 리소스에 대한 로컬 보기를 제공하는 방법은 네임스페이스를 통하는 것이다"**:
    - 리눅스 네임스페이스는 **프로세스가 볼 수 있는 리소스의 범위를 제한**하여 마치 프로세스가 고유한 환경에서 실행되는 것처럼 보이게 합니다.
    - 네임스페이스의 주요 종류:
        1. **pid namespace**: 프로세스 ID를 격리하여 다른 네임스페이스의 프로세스를 보지 못하게 함.
        2. **net namespace**: 네트워크 인터페이스를 격리하여 각 네임스페이스에 독립된 네트워크 스택 제공.
        3. **mnt namespace**: 마운트된 파일 시스템을 격리하여 각 네임스페이스가 별도의 디렉터리 구조를 가질 수 있음.
        4. **uts namespace**: 호스트 이름과 도메인 이름을 격리하여 각 네임스페이스에 독립적인 시스템 이름을 제공.
        5. **user namespace**: 사용자 및 권한을 격리하여 서로 다른 네임스페이스의 사용자 계정이 독립적이도록 함.
    - 네임스페이스를 통해 **각 프로세스는 자신에게만 보이는 로컬 환경**을 가질 수 있게 되며, 이는 가상화와 유사한 환경을 제공합니다.

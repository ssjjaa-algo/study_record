# 2. nginX & Docker

- docker run -d -p 8080:80 --name nginx-8080 nginx
    - -d 옵션은 백그라운드에서 실행한다는 의미.
    - 주로 서버 애플리케이션을 실행할 때 사용
    - docker ps로 Status가 up인지 확인한다.
- docker logs
    - 로그 확인

- -p 옵션
    - 도커는 독립적인 네트워크 인터페이스
    - 이 네트워크 인터페이스의 특정 포트와 호스트의 특정 포트를 열어서
    - 컨테이너에서 동작하는 서버 애플리케이션이 **`외부와 통신하도록 설정`**
    - -p 8080:80
        - 8080은 호스트로 연결할 포트 번호.
        - :(콜론) 문자 뒤의 80은 컨테이너 내부 포트
        - 컨테이너에서 80포트를 지정해서 서버를 실행하면
            - 호스트의 8080 포트로 연결된다.

## HTML 페이지 서빙

- nginX를 미들웨어의 기능으로 사용하기 위한
    - 서버를 구성할 때 제일 앞단에서 트래픽을 받아 전처리를 하거나
    - 적절한 백엔드 서버로 트래픽을 전송
- 새로운 작업 디렉터리를 만들고
    - html, css, js.. 파일들을 구성해서 넣으면 웹 사이트 서빙.

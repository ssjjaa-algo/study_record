# Service

- 클러스터에서 실행 중인 애플리케이션을 단일 외부 엔드포인트를 통해 노출시킨다

- 일부 파드 집합(이를 "백엔드"라고 합시다)이 클러스터 내 다른 파드(이를 "프런트엔드"라고 합시다)에 기능을 제공하는 경우, 프런트엔드는 백엔드 워크로드의 IP 주소를 어떻게 찾고 추적하여 연결할 수 있을까?
    - 이 문제를 해결하기 위해 서비스가 등장

# Service API

- 파드 그룹을 네트워크 상에 노출할 수 있도록 도와주는 추상화
- 결합 분리
    - 프론트엔드 입장에서 백엔드 파드랑 통신이 필요한데, 백엔드 파드 전체를 알 필요 없다. 그냥 추상화임

## 코드

```yaml
apiVersion: v1  # 사용할 API 버전을 정의합니다. 여기서는 v1을 사용합니다.
kind: Service   # 이 객체의 종류가 "Service"임을 나타냅니다.
metadata:
  name: my-service  # 서비스의 이름을 지정합니다. 여기서는 "my-service"입니다.
spec:
  selector:  # 서비스가 타겟팅할 파드를 찾기 위한 레이블 선택자를 정의합니다.
    app.kubernetes.io/name: MyApp  # 레이블 키와 값입니다. 이 레이블이 지정된 파드가 서비스의 대상이 됩니다.
  ports:  # 서비스가 노출할 포트를 정의하는 섹션입니다.
    - protocol: TCP  # 사용할 프로토콜을 지정합니다. 여기서는 TCP를 사용합니다.
      port: 80  # 서비스가 클러스터 내에서 사용할 포트입니다. 이 포트로 서비스가 외부에 노출됩니다.
      targetPort: 9376  # 선택된 파드에서의 실제 포트입니다. 즉, 파드가 수신하는 포트입니다.

```

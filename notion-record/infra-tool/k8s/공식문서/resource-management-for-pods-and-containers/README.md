# Resource Management for Pods and Containers

- Pod을 지정할 때 각 컨테이너가 필요로 하는 리소스를 선택적으로 지정
- 가장 일반적으로 지정되는 리소스는 CPU와 메모리(RAM)
- Pod 내 컨테이너에 대한 리소스 요청을 지정하면, kube-scheduler는 이 정보를 사용하여 해당 Pod를 어떤 노드에 배치할지 결정
- 컨테이너에 대한 리소스 제한을 지정하면, kubelet은 해당 컨테이너가 설정한 제한 이상으로 해당 리소스를 사용하지 않도록 그 한도를 강제
    - kubelet은 최소한 요청된 양만큼의 시스템 리소스를 해당 컨테이너가 사용할 수 있도록 예약

- 리소스 요청은 "이만큼은 무조건 필요해!"라고 요구하는 최소한의 자원
- 리소스 제한은 "이만큼까지만 쓸 수 있어!"라고 설정하는 최대 자원
    - 노드에 여유가 있다면, 요청보다 더 많이 사용할 수 있다
    - 제한을 넘으면 시스템에서 강제로 사용을 막는다.

## Pod의 리소스 요청과 제한

- 리소스 요청과 제한은 개별 **컨테이너**에 대해 설정되지만, 전체 **Pod**의 리소스 요청 및 제한도 고려하는 것이 중요
- Pod의 특정 리소스에 대한 요청/제한은 그 Pod에 포함된 **모든 컨테이너의 해당 리소스 요청/제한을 더한 값**
- 각 컨테이너마다 리소스를 설정하더라도, 최종적으로 Pod 단위에서 리소스를 할당할 때는 모든 컨테이너가 요청하거나 제한된 리소스를 **`합산해서`** Kubernetes가 해당 Pod에 적절한 노드를 할당

### 예시

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: container-1
    image: nginx
    resources:
      requests:
        memory: "256Mi"
        cpu: "500m"
      limits:
        memory: "512Mi"
        cpu: "1"
  - name: container-2
    image: redis
    resources:
      requests:
        memory: "128Mi"
        cpu: "250m"
      limits:
        memory: "256Mi"
        cpu: "0.5" # 500m과 동일, milliCpu 형식 사용이 권장(실수 방지를 위해)
```

- 이 Pod는 실행되기 위해 최소 **384MiB의 메모리**와 **750m의 CPU** 리소스를 요청
- 이 Pod는 최대 **768MiB의 메모리**와 **1.5 CPU**를 사용

## CPU 리소스 단위

- **1 CPU**는 **1 물리적 CPU 코어** 또는 **1 가상 코어**
- 소수점 단위 요청 가능
- 1ms 이하 설정 불가
- 컨테이너가 **싱글 코어, 듀얼 코어, 또는 48코어 머신**에서 실행되더라도 **거의 동일한 계산 능력을 나타냄**
    - 500ms → 0.5cpu

## 메모리 리소스 단위

- `128974848` (바이트)
- `129e6` (129 * 10^6 바이트)
- `129M` (메가바이트)
- `128974848000m` (밀리바이트)
- `123Mi` (미비바이트)

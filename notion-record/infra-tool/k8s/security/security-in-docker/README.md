# Security in Docker

- 다수의 운영체제 프로세스
- Docker 디먼, SSH 서버 등으로 실행되는 프로세스 집합을 가지고 있다.
- Ubuntu Docker 컨테이너를 실행했다고 하자.
    - 컨테이너는 호스트로부터 완전히 격리되어 있지 않다.
    - 컨테이너와 호스트가 같은 커널을 공유한다.
        - Linux에서 네임스페이스를 이용해서 격리
        - 호스트에서 실행되지만 네임스페스에 의해 격리
        - 고유의 네임스페이스
            - 즉 시스템 내에서 컨테이너를 격리하는 방법으로 **`네임스페이스`**를 사용

- Docker는 루트 사용자와 그 외 사용자를 가지고 있음
- 루트 사용자로 실행되는 것을 원하지 않는다면 새 사용자 ID를 지정
- 이미지 자체를 건드려서 USER를 다른 사람으로 설정할 수도 있음 → 이 방법이 간편할 듯.
    - dockerfile 만들 때

- 호스트 root / 컨테이너 root

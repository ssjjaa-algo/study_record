# 2장

- IaC(Infrastructure as Code)
    - YAML, JSON 형식으로 작성한 선언적 코드를 통해 배포하는 컨테이너로 주변 리소스를 관리

## 스케일링 / 오토스케일링

- 컨테이너 클러스터를 구성하여 여러 쿠버네티스 노드 관리
    - 쿠버네티스 위에 동일한 컨테이너 이미지의 레플리카 배포
        - 부하 분산 / 오토스케일링

## 스케줄링

- 배포 위치 결정
- Affinity / Anti-Affinity
    - ex ) Disk I/O가 많은 컨테이너를 디스크가 SSD인 k8s 노드에 배치한다

## 리소스 관리

- 특별한 배치 지정이 없는 경우 CPU나 메모리의 여유 리소스 상태에 따라 스케줄링된다

## 자동화된 복구

- 프로세스 정지 감지 → 컨테이너 스케줄링을 통해 자동 배포

## 로드 밸런싱과 서비스 디스커버리

- 로그 밸런서 기능(Service / Ingress)
    - 라우팅
- 컨테이너를 확장할 때 엔드포인트가 되는 서비스에 컨테이너의 자동 등록과 삭제, 컨테이너 장애 시 분리, 롤링 업데이트 시 필요한 사전 분리 작업 등 자동으로 실행
- 서비스 디스커버리
    - 유레카 디스커버리랑 비슷한 개념인 듯

## 데이터 관리

- 백엔드 데이터 스토어(etcd)
    - 클러스터를 구성한 이중화 가능
    - manifest 파일 (yaml, json..) 파일도 이중화 구조로 저장
    - 설정 파일이나 인증 정보 등의 데이터를 저장하는 구조도 가지고 있음

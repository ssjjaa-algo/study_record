# kube-proxy

- 쿠버네티스 클러스터에서 모든 파드는 서로 통신할 수 있다
    - 클러스터에 배포된 **`파드 네트워킹 솔루션`** 덕분
- 파드 네트워크
    - 클러스터 내의 모든 노드에 걸쳐 있는 가상 네트워크
    - 모든 파드는 이 네트워크에 연결되어 서로 소통
    - 첫 번째 노드에 웹 애플리케이션, 두 번째 노드에 데이터베이스
    - 파드의 IP는 고정적이지 않음 → 더 좋은 방법은 Service를 이용하는 것
        - Service가 파드로 요청 전달해주는 것

## kube-proxy

- 각 노드에서 실행되는 프로세스
- 새로운 서비스가 생성되는지 지속적으로 감시
- 서비스가 생성될 때마다 각 노드에 네트워크 규칙을 만든다
    - 서비스 IP로 향하는 트래픽을 실제 백엔드 파드로 전달 (주로 iptables 이용)

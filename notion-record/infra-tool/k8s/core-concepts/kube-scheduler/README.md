# kube-scheduler

- 어떤 파드가, 어떤 노드에 들어가야 하는지만 결정한다
- 파드를 노드에 두는 것이 아니라, 어디에 들어가야 할 지만 결정
- 파드를 노드에 배치하는 작업은 kublet의 역할이다
- 적절한 컨테이너가 적절한 배에 실리기를 원하듯이 스케줄링 작업이 필요하다

## 스케줄러가 적합한 노드를 찾기 위한 두 단계

- filtering
    - 해당 파드의 프로필에 맞지 않은 노드 걸러내기
        - 파드가 요청한 CPU나 메모리 리소스가 부족한 노드 걸러내기 등
- Scoring
    - 필터링을 통과한 노드들 중에서 가장 적합한 노드를 선택하기 위한 순위 매기기
    - 우선순위 함수를 써서 0~10점 사이 부여
        - 예를 들어 파드 배치 후 노드에 남게 될 여유 리소스를 계산
        - 이 때는 여유 리소스가 더 많이 남는 노드가 더 높은 점수를 얻는다
- 자신 만의 스케줄러도 직접 작성할 수 있다.
    - 리소스 요구 사항
    - 제한
    - Taints / Tolerations
    - Node selectors
    - Affinity
    - 등등

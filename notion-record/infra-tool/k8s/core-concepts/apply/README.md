# apply

## apply

- 로컬 설정 파일
- k8s에 존재하는 실제(live) 객체 정의
- 마지막으로 적용된 설정(last applied configuration)
- 이 세 가지를 모두 고려하여 어떤 변경을 적용할지 결정한다

### 객체가 생성될 때

- 로컬에서 작성한 설정 파일과 유사한 객체 설정이 k8s 내부에 생성됨
- 여기에 객체의 상태를 저장하기 위한 추가 필드들이 포함됨
- 이게 실제 live 객체
- apply 명령을 사용할 경우
    - yaml 파일이 json 형식으로 변환되어
    - 마지막으로 적용된 설정으로 저장된다

### 예시

- 로컬 파일 이미지 변경 시
    - 실제(live) 설정에 있는 값과 비교하여 차이가 있다면 설정이 새 값으로 업데이트됨

### 마지막 적용 설정(JSON)은 어디에 저장되는가

- k8s 클러스터 내 실제 객체 설정에 annotation 형태로 저장됨
    - last applied configuration
- **kubectl apply 명령어를 사용할 때만 수행된다**
    - 따라서 명령형 방식과 선언형 방식을 섞어서 사용하지 말아야 한다

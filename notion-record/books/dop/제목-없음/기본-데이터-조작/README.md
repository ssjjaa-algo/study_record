# 기본 데이터 조작

상태: 시작 전

- DOP는 데이터 모델을 맵과 배열의 유연한 조합으로 표현하고 각 정보 요소를 정보 경로를 통해 접근할 수 있도록 규정한다
- 레코드를 문자열 맵으로 표현하면 시스템이 유연해진다
- 데이터는 항목을 추가하고 삭제하고 필드 명칭을 변경하는 범용 함수 덕분에 **`일급 시민`**이 된다
- 코드는 조작해야 하는 레코드 안의 특정 필드의 키만 알면 된다
- 코도는 레코드의 모든 키를 몰라도 상관 없다 (관련있는 키만 알면 된다)

## 데이터 모델 설계

- Library, Catalog, Book 같은 것은 데이터 개체이다
    - 레코드라고 부른다
- 데이터 개체도는 레코드로 구성된다
    - 레코드의 값은 원시 자료형, 순차 컬렉션, 색인 중 하나이다.
- 데이터 개체 간의 관계는 포함과 연관 중 하나이다
    - 두 관계 유형 모두 일대일, 일대다, 다대다 중 하나일 수 있다

## 맵으로 관리되는 레코드

- 코드는 데이터를 명시적인 인자로 전달받는 정적 함수로 구성된다
- 데이터 개체는 레코드로 모델링되며 레코드 간의 관계는 순차 컬렉션과 색인으로 표현된다

# 9장 옵티마이저와 힌트

## 쿼리 실행 절차

MySQL 서버에서 쿼리가 실행되는 과정을 세 단계로 구분한다.

1. 사용자로부터 요청된 SQL을 쪼개서 MySQL 서버가 이해할 수 있는 수준으로 트리를 구성한다.

SQL 파싱이라고 표현하며, 컴파일러의 역할과 비슷하다고 이해하면 되겠다. 

1. 만들어진 트리를 기반으로 어떤 테이블부터 읽고 어떤 인덱스를 이용해 테이블을 읽을지 선택한다.

불필요한 조건과 연산을 단순화하고, 조인의 경우 어떤 순서로 읽을지, 인덱스 통계 정보를 이용해 사용할 인덱스를 결정하는 등의 작업을 한다. 즉 **“최적화 및 실행 계획 수립”** 단계이며, 옵티마이저가 하은 일이다. 이 단계를 완료하면 **실행 계획**이 만들어진다.

1. 두 번째 단계에서 결정된 정보를 기반으로 스토리지 엔진에게 데이터를 가져오라고 요청한다.

실행 계획에 따라 스토리지 엔진에게 일을 위임하는 것이다. 스토리지 엔진이 결과를 가져오면, MySQL 엔진이 받은 레코드를 조인하거나 정렬하는 등의 작업을 수행한다.
